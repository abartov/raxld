Interedition OAC server

A prototype implementation

Getting started

  1. Store annotation body and get a URI for it [optional]

    POST JSON to http://interedition.performantsoftware.com/annotation_bodies like this one:
    { "mime_type" : "text/html", "content" : "This is a <b>bold</b> annotation!" }
    You will receive a 302 response redirecting you to a freshly-minted URI for it.

  2. Store a new annotation and get a URI for it

    POST JSON to http://interedition.performantsoftware.com/annotations like the following:
    { "author_uri" : "http://people.org/asaf_bartov", "body_uri" : "http://interedition.performantsoftware.com/annotation_bodies/1", "targets" : [ { "uri" : "http://example.com/texts/Alice_in_Wonderland", "constraint" : { "constraint_type" : "xpath", "constraint" : "//TEI.2/foo/bar" } } ] }
    (if your annotation body is not already URI-addressable, use annotation body URI from previous step

  3. Retrieve an annotation

    GET http://interedition.performantsoftware.com/annotations/1 with curl with an "Accept: application/json" header to get the first annotation etc.
    You will get JSON like:
    {"annotation":{"annotation_body":{"uri":"http://interedition.performantsoftware.com/annotation_bodies/1","mime_type":"text/html","created_at":"2011-10-09T12:07:50Z","updated_at":"2011-10-09T12:08:51Z"},"annotation_target_instances":[{"annotation_target_instance":{"annotation_constraint":{"constraint_type" : "marco's", "constraint" : "{ "position":"140-147","checksum":"fee1dead","context":"some useful buffer for diff-like contextualizing"}","created_at":"2011-10-09T12:08:51Z","updated_at":"2011-10-09T12:08:51Z"},"annotation_target_info":{"uri":"http://example.com/texts/Alice_in_Wonderland","mime_type":null}}}]}}

  4. Query available annotations for a target URI: GET /annotations/query?q=your_target_uri_here

    and Accept: application/json. You'll get a JSON array of annotations that target it. Then you can look up the annotations individually with a GET request for this URIs, with an Accept: application/json header.

  5. Render a TEI document into HTML with any annotations stored on this server for it: GET /annotations/render_annotated_tei?uri=your_TEI_document_here

    an XHTML rendering of the TEI document will be returned, with any known annotations inserted according to the XPath constraints stored in the annotations. Note that the annotations are currently assumed to be valid XPath expressions that reflect an existing position in the TEI file.

Additional Functions

  • Update an annotation body

    PUT http://interedition.performantsoftware.com/annotation_bodies/1 with curl -X PUT and a "Content-Type: application/json" header, and the updated JSON object in the body. JSON must look like the JSON returned by GETting the annotation body.

  • Create Geonames-based annotations for a list of tokens and target URIs

    POST http://--Current-IP--/annotations/annotate_with_geonames with a plaintext body of TAB-separated values.

    The expected file format is:

    <token> <TAB> <document-URI> <TAB> <XPath1,frequency> <TAB> <XPath2,frequency> ...

    If you Accept: application/json, a list of URIs for the annotations created will be returned. Otherwise, an HTML page with an annotation count and a list of URIs is returned.
